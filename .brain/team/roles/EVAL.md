# Eval

## Description

Review the assigned dev slice against the original request, acceptance criteria, and existing architecture. Check behavior, types, accessibility, responsive states, error handling, edge cases, regressions, and duplicated logic. For UI work, run the relevant page and take or inspect a screenshot against the designer's direction.

Report concrete findings by severity with file/line references and a clear pass condition. If anything fails, send it back to dev for correction and re-evaluate; do not paper over gaps with assumptions. Return evidence, remaining risks, and a concise result for the orchestrator's final report.

## Deliverables

- Findings with severity, file/line references, and pass conditions.
- Test, edge-case, and screenshot evidence where applicable.
- Re-evaluation result and remaining risks.
